# 记忆系统设计

## 概述

本文档描述 genban-py 项目中记忆系统的实现设计与分阶段任务拆解。
该系统设计参考了：
1、MIRIX记忆分层理念：https://github.com/Mirix-AI/MIRIX
2、memU主动记忆生命周期设计：https://github.com/NevaMind-AI/memU
3、EverMemOS记忆系统架构设计：https://github.com/EverMind-AI/EverMemOS

---

## 架构设计

架构图可参考文件 `docs/架构设计/260225记忆系统架构图.drawio.png`
记忆系统采用**分层 + 切面**的架构设计：

- **分层**：记忆数据从原始对话逐层提炼为抽象记忆
- **切面**：通过监听切面无侵入地接入对话流程

### 核心组件

| 组件         | 说明                                     |
| ---------- | -------------------------------------- |
| **对话监听切面** | 无侵入地监听对话流程，包含对话持久化、上下文管控、提示词注入、记忆召回等能力 |
| **记忆构建**   | 分层构建记忆：初始写入 → 总结概述 → 提取抽象              |
| **记忆存储**   | 三种记忆采用不同存储方式                           |
| **记忆检索**   | 支持基于语义的记忆检索，提供不同场景所需的记忆召回能力            |

### 核心实体类

| 实体类                     | 说明             | 存储方式       | 特点           |
| ----------------------- | -------------- | ---------- | ------------ |
| **Chat** 对话内容           | 对话原始数据信息       | 文件存储       | 原始数据         |
| **ChatOutline** 对话概述    | 对每段对话的简练概述     | 数据库存储      | 时序数据         |
| **CoreMemory** 核心记忆     | 用户画像、高频使用的稳定信息 | 文件存储       | 稳定、高频使用      |
| **SemanticMemory** 语义记忆 | 关键事实、知识点       | 文件存储       | key-value 结构 |
| **EpisodicMemory** 情景记忆 | 何时何地发生了什么事     | 向量存储+数据库存储 | 支持语义检索+时间检索  |
