# 对外接口文档

## 基础信息

- **Base URL**: `http://localhost:8000`
- **接口前缀**: `/api`

---

## 1. 用户登录

### 接口描述
用户登录，获取访问令牌

### 请求信息

**方法**: `POST`  
**路径**: `/api/login`  
**Content-Type**: `application/json`

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user_id | `string` | 是 | 用户ID |
| password | `string` | 是 | 密码 |

### 请求示例

```bash
curl -X POST "http://localhost:8000/api/login" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "alice",
    "password": "123456"
  }'
```

### 响应示例

**成功**:
```json
{
  "token": "abcd1234...",
  "expires_at": 1700000000,
  "error": ""
}
```

**失败**:
```json
{
  "token": "",
  "expires_at": 0,
  "error": "User not found"
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| token | `string` | 访问令牌，有效期7天 |
| expires_at | `integer` | 过期时间（秒级时间戳） |
| error | `string` | 错误信息，成功为空 |

## 2. 流式对话

### 接口描述
发送消息并流式接收 AI 响应（Server-Sent Events）。支持多轮大模型调用和工具调用，可能返回 `assistant` 和 `tool` 两种角色的消息。

### 请求信息

**方法**: `POST`  
**路径**: `/api/talk`  
**Content-Type**: `application/json`  
**认证**: `Authorization: Bearer <token>`

### 请求头

| 头信息 | 必填 | 说明 |
|--------|------|------|
| Authorization | 是 | Bearer Token，格式：`Bearer <token>` |

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user_input | `string` | 是 | 用户输入内容 |

### 请求示例

```bash
curl -X POST "http://localhost:8000/api/talk" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer abcd1234..." \
  -d '{
    "user_input": "你好，请介绍一下自己"
  }'
```

### 响应格式

**Content-Type**: `text/event-stream`

响应采用 SSE (Server-Sent Events) 格式，包含以下三种事件类型：

#### 1. message 事件

流式返回 AI 的回答内容，每次返回全量内容。根据消息角色不同，返回的字段有所区别。

**事件格式 - assistant 角色**:
```
event: message
data: {"source": "assistant", "id": "msg_xxx", "role": "assistant", "content": "回答内容", "reasoning_content": "思考过程", "tool_calls": null}
```

**事件格式 - tool 角色**:
```
event: message
data: {"source": "tool", "id": "msg_xxx", "role": "tool", "content": "工具调用结果", "reasoning_content": "", "tool_calls": null}
```

**响应字段说明**:

| 字段 | 类型 | 说明 | 适用角色 |
|------|------|------|----------|
| source | `string` | 消息来源，可能为 `assistant`, `tool`, `command` | 所有 |
| id | `string` | 消息唯一标识 | 所有 |
| role | `string` | 消息角色，可能为 `assistant`, `tool` | 所有 |
| content | `string` | AI 的回答内容或工具调用结果（全量） | 所有 |
| reasoning_content | `string` | AI 的思考过程内容（全量） | 仅 `assistant` |
| tool_calls | `list` | 工具调用信息 | 仅 `assistant` |

#### 2. complete 事件

当 AI 回答完成时触发，表示流式响应结束。

**事件格式**:
```
event: complete
data: 
```

#### 3. error 事件

当接口发生错误时触发。

**事件格式**:
```
event: error
data: {"error": "错误描述信息"}
```

### 错误响应

**401 未认证**:
```json
{
  "detail": "Unauthorized"
}
```

### 响应示例

#### 示例 1：普通对话（无工具调用）

```
event: message
data: {"source": "assistant", "id": "msg_001", "role": "assistant", "content": "", "reasoning_content": "让我", "tool_calls": null}

event: message
data: {"source": "assistant", "id": "msg_001", "role": "assistant", "content": "", "reasoning_content": "让我思考", "tool_calls": null}

event: message
data: {"source": "assistant", "id": "msg_001", "role": "assistant", "content": "", "reasoning_content": "让我思考一下...", "tool_calls": null}

event: message
data: {"source": "assistant", "id": "msg_001", "role": "assistant", "content": "你好", "reasoning_content": "让我思考一下...", "tool_calls": null}

event: message
data: {"source": "assistant", "id": "msg_001", "role": "assistant", "content": "你好！", "reasoning_content": "让我思考一下...", "tool_calls": null}

event: complete
data: 
```

#### 示例 2：复杂对话（包含命令调用和工具调用）

```
event: message
data: {"source": "command", "id": "msg_000", "role": "assistant", "content": "系统状态：正常", "reasoning_content": "", "tool_calls": null}

event: message
data: {"source": "assistant", "id": "msg_001", "role": "assistant", "content": "", "reasoning_content": "用户想查询天气", "tool_calls": null}

event: message
data: {"source": "assistant", "id": "msg_001", "role": "assistant", "content": "", "reasoning_content": "用户想查询天气，我需要调用天气工具", "tool_calls": null}

event: message
data: {"source": "assistant", "id": "msg_001", "role": "assistant", "content": "我来为您查询天气", "reasoning_content": "用户想查询天气，我需要调用天气工具", "tool_calls": [{"function": {"name": "get_weather", "arguments": "{\"location\": \"北京\"}"}}]}

event: message
data: {"source": "tool", "id": "msg_002", "role": "tool", "content": "北京今天晴，25°C", "reasoning_content": "", "tool_calls": null}

event: message
data: {"source": "assistant", "id": "msg_003", "role": "assistant", "content": "", "reasoning_content": "工具返回了天气信息", "tool_calls": null}

event: message
data: {"source": "assistant", "id": "msg_003", "role": "assistant", "content": "北京今天天气晴朗，气温25°C", "reasoning_content": "工具返回了天气信息", "tool_calls": null}

event: complete
data: 
```

### 注意事项

1. **认证**: `/talk` 接口需要在请求头中携带 `Authorization: Bearer <token>`
2. **Token 获取**: 通过 `/login` 接口获取 token，有效期7天
3. **全量返回**: 每次 `message` 事件返回的 `content` 和 `reasoning_content` 都是**全量内容**，前端直接替换显示即可，无需拼接
4. **多角色消息**: 接口可能返回 `role="assistant"` 和 `role="tool"` 两种角色的消息，前端需要根据角色进行不同的展示处理
5. **工具调用流程**: 当 `assistant` 消息触发工具调用时，会依次返回工具调用结果（`role="tool"`），然后继续返回 `assistant` 的最终回复
6. **多轮调用**: 单次 `/talk` 调用可能涉及多轮大模型调用（工具调用场景），前端需要持续监听直到收到 `complete` 事件
7. **错误处理**: 需要监听 `error` 事件处理异常情况，同时注意 401 认证失败响应
8. **完成标识**: 通过 `complete` 事件判断流式响应结束
