# 用户模块设计

## 概述

本文档描述 genban-py 项目中用户模块的设计方案，包括用户数据管理、登录认证和接口校验。

---

## 需求背景

当前系统使用环境变量中的默认用户 ID，没有用户认证机制。需要引入用户管理模块，实现：
1. 用户数据的持久化存储（SQLite）
2. 用户登录认证
3. 接口访问权限校验

---

## 数据表设计

### 用户表 (users)

| 字段名 | 类型 | 说明 |
|-------|------|------|
| userId | TEXT PRIMARY KEY | 用户唯一标识 |
| passwordHash | TEXT | 密码哈希值 |
| createdAt | INTEGER | 创建时间（秒级时间戳） |

## 认证机制

### Token 方案

采用简单的随机 Token 机制：

1. **登录成功**：生成随机字符串作为 Token，返回给客户端
2. **Token 存储**：服务端内存中维护 `token -> userId` 的映射
3. **Token 有效期**：默认 7 天，过期后需要重新登录
4. **Token 校验**：从请求 Header 中提取 Token 进行校验

## 接口设计

### 1. 登录接口

**POST** `/api/login`

**请求体：**
```json
{
    "userId": "string",
    "password": "string"
}
```

**响应成功 (200)：**
```json
{
    "token": "random_token_string",
    "expiresAt": 1700000000
}
```

**响应失败 (401)：**
```json
{
    "error": "Invalid credentials"
}
```

### 2. Talk 接口（增加校验）

**POST** `/api/talk`

**请求头：**
```
Authorization: Bearer <token>
```

**请求体：**
```json
{
    "userInput": "string"
}
```

**校验失败 (401)：**
```json
{
    "error": "Unauthorized"
}
```

|-------|------|
| `UserNotFoundException` | 用户不存在 |

| `UnauthorizedException` | Token 无效或过期 |

---

## 安全考虑

1. **密码存储**：使用 bcrypt 进行哈希，不存储明文
2. **Token 传输**：通过 Header 传输，避免暴露在 URL 中
3. **Token 有效期**：设置 7 天过期，减少被盗用风险
4. **SQL 注入**：使用参数化查询，避免拼接 SQL
