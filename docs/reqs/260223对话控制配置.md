# 对话控制配置 - 需求分析和技术实现方案

## 需求背景

为 genban-py 项目新增对话框控制能力，用户可以通过特定指令在对话过程中：
- 开启/关闭深度思考功能
- 切换当前使用的 AI 模型
- 查看当前会话配置状态

## 需求详情

### 功能需求

1. **深度思考控制**
   - 支持通过指令开启深度思考
   - 支持通过指令关闭深度思考
   - 默认状态：关闭

2. **模型切换**
   - 支持通过指令切换模型
   - 切换后保留历史对话上下文
   - 默认模型：配置列表中的第一个模型

3. **状态查看**
   - 支持查看当前会话的配置状态
   - 显示当前模型和深度思考开关状态

4. **指令格式**
   - 支持英文和中文指令
   - 严格匹配指令格式

### 指令列表

| 指令 | 别名 | 功能 |
|------|------|------|
| `/openthink` | `/开启思考` | 开启深度思考 |
| `/closethink` | `/关闭思考` | 关闭深度思考 |
| `/model:<name>` | `/切换模型:<name>` | 切换至指定模型 |
| `/status` | `/状态` | 查看当前配置状态 |

### 非功能需求

1. **作用范围**：仅当前会话有效，新会话回归默认设置
2. **响应方式**：
   - 指令执行成功：静默处理（`/status` 除外，需返回状态信息）
   - 指令执行失败：抛出异常，由接口层处理并告知前端
3. **架构设计**：指令逻辑内聚，与 Core 松耦合

## 技术实现方案

### 架构设计

采用"指令解析在 Kernel.Core 层"的方案，指令逻辑独立封装：

```
用户输入
    ↓
Core.talk()
    ↓
Commands 模块处理
    ↓
返回 (是否为指令, 处理结果)
    ↓
是指令 → 返回执行结果给前端
非指令 → 继续原有对话流程
```

### Core 与 Commands 交互协议

Core 调用 Commands 模块时传入会话对象和用户输入，Commands 返回元组：
- 元素1（是否为指令）：布尔值
- 元素2（处理结果）：指令执行结果或处理后的用户输入

### 模块职责

#### SessionManager
扩大会话存储结构，增加状态字段用于存储当前会话的配置信息。

#### TalkRequest
移除模型字段，模型由会话状态统一管理。

#### Kernel.Core
作为指令处理的入口，调用 Commands 模块，根据返回结果决定后续流程：
- 若是指令：将执行结果返回给前端
- 若非指令：将处理后的输入继续原有对话流程

#### Kernel.Core.Commands（新增模块）
内聚所有指令相关的解析和执行逻辑，对外提供统一接口。内部实现指令识别、参数提取、状态更新等功能。

#### Hub
扩展调用接口，支持深度思考参数的传递。

#### ApiDashScope
在调用大模型 API 时传入深度思考参数。

### 目录结构

```
kernel/core/
├── core.py                 # 核心流程，调用 Commands
├── components/
│   └── chat_factory.py     # 消息工厂
└── commands/               # 新增：指令模块
    └── commands.py         # 指令解析和执行逻辑
```

### 默认状态

新会话创建时的默认配置：
- 深度思考：关闭
- 模型：配置列表中的第一个模型

### 状态查看输出格式

`/status` 指令的输出格式示例：
```
当前配置：
- 模型：qwen3-32b
- 深度思考：开启
```

### 异常处理

新增异常类型用于处理指令执行失败的情况：
- 模型不存在异常：当切换至配置列表之外的模型时抛出

## 边界情况处理

1. **模型切换验证**
   - 切换前验证模型是否在配置列表中
   - 验证失败抛出异常，保持原模型不变

2. **新会话初始化**
   - 创建会话时从配置读取默认模型
   - 深度思考默认为关闭状态

3. **指令与正常输入**
   - 严格匹配指令格式，避免误判
   - 非指令内容原样传递给后续流程

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 模型切换后上下文不兼容 | 中 | 保留历史对话，由模型自行处理 |
| 深度思考增加 Token 消耗 | 中 | 默认关闭，用户按需开启 |
| 指令与正常输入混淆 | 低 | 严格匹配指令格式，带前缀设计 |
