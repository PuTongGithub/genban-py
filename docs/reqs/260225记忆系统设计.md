# 记忆系统设计

## 概述

本文档描述 genban-py 项目中记忆系统的实现设计与分阶段任务拆解。
该系统设计参考了：
1、MIRIX记忆分层理念：https://github.com/Mirix-AI/MIRIX
2、memU主动记忆生命周期设计：https://github.com/NevaMind-AI/memU
3、EverMemOS记忆系统架构设计：https://github.com/EverMind-AI/EverMemOS

---

## 架构设计

架构图可参考文件 `docs/reqs/记忆系统架构图.drawio.png`
记忆系统采用**分层 + 模块化**的架构设计：

- **分层**：记忆数据从原始对话逐层提炼为抽象记忆
- **模块化**：将系统划分为了对话管理和记忆模块两个部分，实现记忆与对话的解耦

### 核心组件

| 组件             | 说明                                     |
| -------------- | -------------------------------------- |
| **对话管理-对话持久化** | 触发记忆系统-构建流程，将对话增量写入文件系统                |
| **对话管理-上下文管控** | 管控对话上下文，包括识别话题切换、上下文超限触发总结概述、工具调用总结等能力 |
| **对话管理-提示词注入** | 将核心记忆动态注入到系统提示词中                       |
| **对话管理-记忆召回**  | 主动识别对话意图，触发记忆检索，将记忆写入对话上下文中            |
| **记忆系统-构建**    | 分层构建记忆：初始写入 → 总结概述 → 提取抽象              |
| **记忆系统-存储**    | 三种记忆采用不同存储方式                           |
| **记忆系统-检索**    | 支持基于语义的记忆检索，提供不同场景所需的记忆召回能力            |

### 核心实体类

| 实体类                         | 说明           | 存储方式       | 特点           |
| --------------------------- | ------------ | ---------- | ------------ |
| **Chat** 对话内容               | 对话内容的原始数据    | 文件存储       | 原始数据         |
| **ConversationMemory** 对话记忆 | 对每段对话的简练概述   | 文件存储+数据库存储 | 时序数据         |
| **CoreMemory** 核心记忆         | 稳定且高频使用的画像信息 | 文件存储       | 用户画像、自我画像    |
| **SemanticMemory** 语义记忆     | 关键事实、知识点     | 文件存储       | key-value 结构 |
| **EpisodicMemory** 情景记忆     | 何时何地发生了什么事   | 向量存储+数据库存储 | 支持语义检索+时间检索  |
